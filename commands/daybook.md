# Daybook

Engineering Daybook 작성을 도와주는 서브커맨드입니다.
양식·템플릿·태그 규칙은 노트 루트의 CLAUDE.md에 위임한다.

## 실행 프로토콜

### 전제조건: 노트 루트 경로 확인

`~/.claude/note-plugins.json`에서 `noteRoot` 값을 확인한다. 파일이 없으면 새로 생성한다. `noteRoot`가 없거나 존재하지 않는 경로이면 AskUserQuestion으로 물어보고, 응답을 저장한다.

### Phase 0: 할 일 자동 생성 (업무 시작 시)

1. **전날 daybook 읽기**
   - 전날 디렉터리의 `Daybook.md` 읽기
   - `## 회고` 섹션이 있으면 우선 참고, 없으면 전체 내용 참고
   - 파일이 없으면 빈 초안으로 시작
2. **할 일 초안 생성**
   - 전날 내용을 분석해 오늘 이어서 할 작업을 AI가 추론하여 제안
3. **사용자 확인 및 추가 입력**
   - 초안을 보여주고 AskUserQuestion으로 추가/수정 입력 받기 ("건너뛰기" 옵션 제공)
4. **Daybook.md에 배치**
   - `## 할 일` 섹션을 Daybook.md 최상단에 작성

### Phase 1: 파일 준비

1. 오늘 날짜 디렉터리 확인 (`{DD} {Day}/`)
2. 디렉터리가 없으면 생성하고, `Daybook.md`를 CLAUDE.md 템플릿으로 생성 (Phase 0 결과 포함)
3. 디렉터리가 있으면 `Daybook.md` 읽기

### Phase 2: 파일 작성

사용자가 내용을 전달하면:
1. 개별 섹션 파일(`{주제}.md`)을 생성하고 내용 작성
2. `Daybook.md`의 `## 기록` 섹션에 `![[주제]]` embed 추가
3. 섹션 파일에 `feature` 필드가 있으면, 작성 완료 후 안내한다: "project 문서에 daybook 링크를 추가하려면 `/note-plugins:project`를 실행하세요"

### Phase 3: Git 커밋 (선택)

작성 완료 후 커밋 여부 확인.

## 주의

- 변경 파일 목록은 기록하지 않는다. git이 관리할 일이다.
- 구조화된 질문으로 사용자를 유도하지 않는다.
