# 스킬 구조 평가 체크리스트

공식 best practices 기반. 각 항목을 O/X/해당없음으로 판정한다.

## description

| # | 항목 | 판정 기준 |
|---|------|-----------|
| 1 | "무엇을 하는지" 포함 | 스킬의 기능이 명확히 기술되어 있는가 |
| 2 | "언제 쓰는지" 포함 | 트리거 조건/맥락이 기술되어 있는가 |
| 3 | 3인칭으로 작성 | 1인칭("I can..."), 2인칭("You can...") 사용하지 않는가 |
| 4 | 구체적 키워드 포함 | 모호한 표현("helps with stuff") 없이 핵심 키워드가 있는가 |
| 5 | 1024자 이하 | 최대 길이 초과하지 않는가 |

## frontmatter

| # | 항목 | 판정 기준 |
|---|------|-----------|
| 6 | name 규칙 준수 | 소문자+하이픈, 64자 이하, 예약어(anthropic, claude) 미포함 |
| 7 | disable-model-invocation 판단 적절 | 부작용 있는 작업(배포, 커밋, 메시지 전송)은 true인가 |
| 8 | allowed-tools 지정 | 필요한 도구만 명시했는가 (특히 서브에이전트 실행 시 중요) |
| 9 | argument-hint 지정 | 인자를 받는 스킬이면 힌트가 있는가 |

## SKILL.md 본문

| # | 항목 | 판정 기준 |
|---|------|-----------|
| 10 | 500줄 이하 | 초과 시 참조 파일로 분리했는가 |
| 11 | 간결함 | Claude가 이미 아는 내용을 장황하게 설명하지 않는가 |
| 12 | 시간 의존 정보 없음 | "2025년 이후엔..." 같은 표현이 없는가 |
| 13 | 용어 일관성 | 같은 개념에 다른 용어를 혼용하지 않는가 |
| 14 | 선택지 과다 없음 | 기본 하나 + 대안 하나 이내인가 |
| 15 | Windows 경로 없음 | 백슬래시(`\`) 대신 슬래시(`/`) 사용하는가 |
| 16 | 예시가 범용적 | 특정 프로젝트 도메인에 종속되지 않고 다른 코드베이스에도 적용 가능한 예시인가 |

## 참조 파일

| # | 항목 | 판정 기준 |
|---|------|-----------|
| 17 | 참조 깊이 1단계 | SKILL.md → ref.md만 허용. 2단계 이상 중첩 없는가 |
| 18 | 100줄 이상 참조에 목차 | 긴 참조 파일 상단에 구조 요약이 있는가 |
| 19 | SKILL.md에서 참조 안내 | 각 참조 파일의 내용과 읽을 시점이 SKILL.md에 명시되어 있는가 |
