## ⚠️ 이 풀의 본질

**이 풀은 절대로 정답 후보 목록이 아니다.**

- 풀은 *시드 카탈로그* + *발굴 사고가 막혔을 때 참고용*
- **풀 밖에서 발굴한 메타포가 항상 우선 후보**
- 풀 안만 사용한 시안은 LLM 클리셰의 반복 → variant-studio 가치 무효화

LLM은 명시된 목록을 보면 "이 안에서 골라야 한다"고 인식하는 *closed list bias*에 빠진다. 본 스킬에서 가장 위험한 함정. SKILL.md Phase 2 룰을 반드시 따른다.

## 사용 규칙

1. **자유 발굴 먼저** (SKILL.md Phase 2-a) — 풀 보지 말고 도메인·문화·시대·물리 사물에서 메타포 발굴
2. **풀 대조는 마지막** — 자유 발굴 후 시간 절약을 위해 풀 안에 같은 메타포 있는지 체크 (있으면 시그니처 재활용)
3. **N개 시안 중 50% 이상 풀 밖** — SKILL.md 정량 룰
4. **가족 중복 금지** — 스티커 + 폴라로이드는 둘 다 *Paper* 가족 → 함께 뽑지 말 것
5. **풀 밖 신규 메타포는 등록** — 시안 검증 후 이 파일에 추가 (가족·시그니처 3개·발굴 출처 명시)

## 풀 확장 (메타 룰)

**풀 밖에서 발굴한 메타포가 시안으로 검증되면 반드시 등록한다.** 안 하면 풀이 정체되고 다음 세션 사용자가 같은 발굴 비용을 다시 치름.

새 메타포 추가 시:

1. **시그니처 3개 이상 명시** (그 사물이라면 반드시 있는 디테일)
2. **가족 분류** (없으면 새 가족 추가)
3. **발굴 출처 기록** (어느 발굴 루트에서 나왔는지: 도메인·문화·참조·물리·시대·자연)
4. **시안 빌드 검증 후** 등록 — 검증 안 된 메타포는 풀 불순물

### 등록 예

```
| 마법진 | 원형 + 회전 룬 ring + 별빛 + 빛나는 pool | 발굴: 게임 어휘 (메이플 루디브리엄) |
```

### 풀이 진화한다는 의미

- 사용 횟수가 적은 메타포 → 별 표시 또는 archive
- 자주 등장하는 메타포 → 가족 내 정렬 우선순위↑
- 새 가족이 생기면 (예: AR/VR, 신소재) 풀 구조 자체 확장

풀은 *살아있는 자산*. 정적 사전 ❌ 동적 라이브러리 ✅.
