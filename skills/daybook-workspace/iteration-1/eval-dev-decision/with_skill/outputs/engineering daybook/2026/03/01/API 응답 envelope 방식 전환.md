> [!note] Summary
> **배경**
> 기존 API는 데이터를 바로 반환하는 구조였는데, 에러 코드를 표준화하려면 응답 형식을 통일할 필요가 있었다.
>
> **결론**
> 모든 API 응답을 envelope 구조로 감싸기로 결정했다. 성공과 실패 모두 동일한 최상위 구조를 따르도록 한다.

기존에는 API가 도메인 데이터를 그대로 반환했다. 정상 응답에서는 문제가 없지만, 에러가 발생했을 때 응답 형식이 제각각이라 클라이언트에서 일관된 에러 처리가 어려웠다. 에러 코드를 표준화하려면 응답 구조 자체를 통일해야 했다.

envelope 방식으로 전환하면 모든 응답이 동일한 최상위 구조를 갖는다.

```json
{
  "status": "success",
  "data": { ... },
  "error": null
}
```

에러 발생 시에도 같은 구조를 유지한다.

```json
{
  "status": "error",
  "data": null,
  "error": {
    "code": "INVALID_INPUT",
    "message": "..."
  }
}
```

이렇게 하면 클라이언트는 먼저 `status`를 확인하고, 성공이면 `data`를, 실패면 `error`를 읽는 단일 흐름으로 처리할 수 있다.
